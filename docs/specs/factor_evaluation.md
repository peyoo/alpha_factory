模块目标本模块（alpha/evaluation）负责对生成的因子（Feature）进行统计学验证。其核心目标是评估因子的预测能力（IC）、单调性（Quantile Returns）和稳定性（Turnover）。2. 输入数据契约 (Input Data Contract)所有进入评价模块的 DataFrame 必须预先通过 apply_standard_adjustment 处理，确保满足以下 Schema：字段名类型物理意义_DATE_pl.Date交易日期（严格对齐）_ASSET_pl.Categorical资产唯一标识CLOSEpl.Float32后复权收盘价（用于计算收益率）f_*pl.Float32待评价因子列（如 f_rsi_14）3. 核心功能逻辑 (Core Logic)3.1 收益率对齐 (Forward Returns Mapping)为了计算因子性能，必须计算未来 $N$ 日的收益率。公式：$R_{t, t+n} = \frac{CLOSE_{t+n}}{CLOSE_{t}} - 1$实现约束：必须在 _ASSET_ 分组内使用 shift(-n) 操作，严禁跨股票对齐。3.2 截面分层分析 (Quantile Analysis)在每一个交易日（_DATE_）截面上，将资产按因子值大小等分为 $K$ 组。默认组数：5 组或 10 组。Polars 实现：使用 pl.col(f_name).qcut(K).over("_DATE_")。评估指标：计算每组的累积收益曲线，验证是否存在明显的单调性（Q5 > Q4 > ... > Q1）。3.3 信息系数 (IC / Rank IC)衡量因子值与未来收益率的相关性。IC (Pearson)：因子值与未来收益的相关系数。Rank IC (Spearman)：因子值的秩与未来收益的秩的相关系数（对离群值更鲁棒，推荐）。指标扩展：计算 IC 均值、IC 标准差、ICIR (IC Mean / IC Std)。3.4 换手率与衰减 (Turnover & Decay)评估因子的交易成本。自相关性 (Autocorrelation)：因子在 $t$ 时刻与 $t-1$ 时刻的相关性。换手率：各分组内资产变动的比例。4. 影子变量的应用 (Shadowing in Evaluation)在评价模块中，RAW_ 前缀列具有特定用途：流动性过滤：在分析前，利用 RAW_VOLUME == 0 或 RAW_CLOSE == RAW_LIMIT_UP 过滤掉无法买入/卖出的标的。容量评估：利用 RAW_AMOUNT (成交额) 评估因子能够承载的最大资金规模。5. 开发者准则 (Guidelines)Lazy 优先：评价模块应接受 pl.LazyFrame 并返回聚合后的结果，以利用 Polars 的查询优化。多进程加速：对于大规模因子池的分析，应在 alpha/evaluation/engine.py 中使用 concurrent.futures 跨因子并行。可视化解耦：计算逻辑与绘图逻辑分离。计算函数返回统计结果表（Parquet），绘图函数读取结果表并生成 HTML/PNG 报告。
